Задание 13. OAuth → JWT → API
После входа через GitHub откройте страницу с комментариями. Создайте комментарий.

нарисуйте полный flow (текстом или схемой): кнопка → GitHub → callback → сессия → me.php → JWT → React → FastAPI → комментарий.
кнопка - oauth-github (отправляет на GitHub с state (случайная строка), адресом куда возвращаем, айди приложения и данными, которые нам нужны) - GitHub (Авторизация)
- callback (проверяет state, создаёт/находит существующий профиль) - сессия (user_id из callback) - me.php (по куке PHPSESSID генерирует JWT) - React (сохраняет jwt в state)
- FastApi (проверяет jwt и достаёт user_id) - комментарий (по user_id находит имя автора поста и выводит его)

Задание 17. Баги и пакеты
В отчёте: прочитайте раздел «Что вы на самом деле написали» в методичке. Назовите 3 бага в вашем коде, которые пакеты Laravel закрывают из коробки. Для каждого: что за баг, чем опасен, какой пакет решает.
Баг	                Чем опасен	                                                    Как закрывает пакет
Секрет в коде	        Утечёт через git. Боты сканируют GitHub за минуты.	            passport:keys генерирует RSA, хранит в файлах, .gitignore
Нет отзыва токенов	Заблокировали пользователя — JWT час ещё действует.	            Passport: таблица oauth_access_tokens, DELETE
Нет refresh	        Истёк JWT → логиниться заново.	                                    Passport: refresh_token из коробки
Один секрет на всех	Утёк у одного сервиса → скомпрометированы все.	                    Passport: каждый client — свой client_secret
CSRF вручную (state)	Забыл проверить → дыра.	                                            Socialite: проверяет автоматически
Нет scopes	FastAPI может всё. Нет ограничений.	                                    Passport: scope=read-posts, delete-users
Нет валидации exp	Ошибка в time() → токен на 100 лет.	                            Passport: TTL в конфиге, единый
